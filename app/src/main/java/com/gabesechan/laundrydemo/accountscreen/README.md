The accountscreen module is responsible for display the account screen when users are logged in.  The user can edit or delete addresses here (adding a new address is currently not done here, possibly added in the future).  Deleting an address launches a network request in a coroutine.  The current network error corner case is just to tell the user something went wrong.  Logout works by sending a logout request to the server, but immediately logging out locally.  The server request allows it to clear up unnecessary tokens in its db, but we do not wait for it to respond or fail to logout if we can't reach it, we merely delete the local session info and leave the server side session abandoned.
